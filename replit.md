# Fat Finger Event Production Website

## Overview

This is a full-stack web application for Fat Finger, an event production company specializing in festival productions, team outings, corporate parties, business events, and stage hosting. The application showcases their services and portfolio with a modern, visually engaging interface featuring custom animations and a premium design aesthetic.

The application uses a React-based frontend with Vite for development and build tooling, Express for the backend server, and is configured to use PostgreSQL with Drizzle ORM for database operations. The UI is built with shadcn/ui components and Tailwind CSS for styling.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture

**Framework**: React 18+ with TypeScript, using Vite as the build tool and development server.

**Routing**: Client-side routing implemented with Wouter, a lightweight React router. The application has a simple two-page structure (Home and Portfolio) plus a 404 page.

**UI Component System**: Built on shadcn/ui (New York style variant) which provides a comprehensive set of Radix UI primitives with custom styling. This gives access to 40+ pre-built accessible components including dialogs, dropdowns, forms, tables, and more.

**Animation Library**: Framer Motion for page transitions, scroll-based animations, and interactive UI elements. The site features custom animated service cards with different visual themes (ripple effects, confetti, connecting dots, spotlights, grids, and blob animations).

**Styling Approach**: Tailwind CSS v4 (using @import syntax) with custom theme extensions for:
- Custom fonts (Inter for body text, Permanent Marker for display headings)
- Extended color palette with primary accent color (pink/magenta)
- Custom utility classes for glass morphism effects and text shadows
- CSS custom properties for flexible theming

**State Management**: TanStack Query (React Query) for server state management with custom fetch utilities. No global client state management library is used, keeping the architecture simple.

**Image Carousel**: Embla Carousel with autoplay plugin for showcasing portfolio images and partner logos.

### Backend Architecture

**Server Framework**: Express.js with TypeScript, running in ES module mode.

**Development vs Production**: Separate entry points:
- `server/index-dev.ts`: Development server with Vite middleware for HMR and hot reloading
- `server/index-prod.ts`: Production server serving pre-built static assets from dist/public

**API Architecture**: RESTful API structure with all routes prefixed with `/api`. Currently using in-memory storage (MemStorage class) as a placeholder, designed to be easily replaced with database-backed storage.

**Request Logging**: Custom middleware that logs API requests with timing information, truncating long response bodies for readability.

**Static File Serving**: In production, serves the built React application as static files with fallback to index.html for client-side routing.

**Build Process**: Uses esbuild to bundle the production server code, and Vite to build the client application.

### Database Design

**ORM**: Drizzle ORM configured for PostgreSQL via Neon Database's serverless driver.

**Schema Location**: Shared schema definitions in `shared/schema.ts` using Drizzle's table definitions and Zod for validation.

**Current Schema**: Minimal user table with:
- UUID primary key (generated by database)
- Username (unique, required)
- Password (required)

**Validation**: Uses drizzle-zod to create insert schemas from Drizzle table definitions, ensuring type safety between database and application code.

**Migration Strategy**: Drizzle Kit for schema migrations, configured to output to `./migrations` directory. Uses `db:push` script for direct schema synchronization during development.

**Storage Interface**: Abstract IStorage interface allows swapping between in-memory and database-backed implementations without changing application code. Currently implements MemStorage as a placeholder.

- Connection managed through DATABASE_URL environment variable

**Form Handling**:
- React Hook Form for form state management
- Hookform Resolvers for validation integration
- Zod for schema validation

**Styling**:
- Tailwind CSS v4 with @tailwindcss/vite plugin
- Class Variance Authority (CVA) for component variants
- clsx and tailwind-merge for conditional class composition

**Development Tools**:
- Replit-specific plugins for development banner, error overlay, and cartographer
- Custom Vite plugin for updating OpenGraph meta tags with deployment URLs
- TypeScript with strict mode enabled

**Fonts**: Google Fonts (Inter and Permanent Marker) loaded via CDN in index.html.

**Session Management**: Connect-pg-simple for PostgreSQL-backed session storage (configured but not yet implemented in routes).

- Vite for frontend bundling and development
- esbuild for backend bundling
- PostCSS with Tailwind and Autoprefixer
